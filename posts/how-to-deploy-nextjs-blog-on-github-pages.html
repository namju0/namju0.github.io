Next.js 블로그를 GitHub Pages에 배포하는 방법 (Blog Boilerplate 사용)

Next.js Blog Boilerplate (opens in a new tab)를 사용해 생성한 정적 블로그를 GitHub Pages에 배포하는 과정입니다. React 등 기타 요소를 깊숙히 다루지 않고도 간단하고 깔끔한 정적 블로그를 간편하게 제작할 수 있는 방법을 소개합니다.

1. Next.js Blog Boilerplate 클론

1-1. 프로젝트 클론

먼저 Next.js Blog Boilerplate (opens in a new tab)를 로컬에 클론합니다:

git clone <https://github.com/vercel/next.js.git>
cd next.js/examples/blog

1-2. 패키지 설치

클론한 디렉토리에서 패키지를 설치합니다:

npm install

1-3. 로컬 서버 실행

다음 명령어로 로컬 개발 서버를 실행합니다:

npm run dev

브라우저에서 http://localhost:3000에 접속해 블로그가 정상적으로 보이는지 확인하세요.

2. Git 리포지토리 설정

이제 프로젝트를 GitHub 리포지토리에 배포할 수 있도록 Git 리포지토리를 설정합니다.

2-1. Git 리포지토리 생성

GitHub에서 <your-github-username>.github.io 라는 리포지토리를 생성합니다. 생성 시 ‘Initialize this repository with:’ 설정에서 ‘Add a README file’을 체크하여 리포지토리 생성과 동시에 초기화를 합니다.

2-2. GitHub 리포지토리 연결

로컬 프로젝트는 현재 https://github.com/vercel/next.js.git 와 연결되어 있습니다. 로컬 프로젝트를 새로 생성한 GitHub 리포지토리와 연결합니다:

git remote set-url origin https://github.com/<your-github-username>/<your-github-username>.github.io.git

3. GitHub Pages 배포 준비

GitHub Pages에 블로그를 배포하기 위한 몇 가지 추가 설정이 필요합니다.

3-1. gh-pages 패키지 설치

GitHub Pages에 배포하려면 gh-pages 패키지를 설치해야 합니다:

npm install gh-pages --save-dev

3-2. next.config.js 설정

GitHub Pages에서는 리포지토리 이름을 기반으로 경로를 설정해야 합니다. 프로젝트 루트에 있는**next.config.js** 파일을 열어, nextConfig에 다음 내용을 추가 작성합니다:

const nextConfig = {
  // ... 전략 ...
  output: 'export',
  images: {
    unoptimized: true
  }
  // ... 후략 ...
};

3-3. package.json 수정

package.json 파일에 배포를 위한 스크립트를 추가합니다. scripts 섹션을 아래처럼 작성하세요:

{
  "scripts": {
    "dev": "next",
    "build": "node ./scripts/gen-rss.js && next build",
    "start": "next start",
    "deploy": "touch out/.nojekyll && gh-pages -d out --dotfiles"
  }
}

이 스크립트는 npm run build로 out 디렉토리를 생성하고 npm run deploy로 그 디렉토리를 GitHub 리포지토리에 배포할 수 있게 해 줍니다.

4. GitHub Pages 설정 및 배포

4-1. GitHub Pages 설정

GitHub 리포지토리의 Settings > Pages 섹션으로 이동합니다. 배포할 브랜치를 gh-pages로 설정하고, 저장합니다. 그러면 GitHub Pages에서 블로그를 배포할 준비가 완료됩니다.

4-2. 블로그 배포

이제 블로그를 빌드합니다.

npm run build

이 명령어를 실행하면 블로그가 out 디렉토리에 생성됩니다.

이를 GitHub Pages에 배포합니다:

npm run deploy

이 명령어를 실행하면 블로그가 gh-pages 브랜치에 푸시되고 GitHub Pages에 배포됩니다.

4-3. 배포된 블로그 확인

배포가 완료되면 브라우저에서 https://<your-github-username>.github.io로 접속하여 블로그가 정상적으로 배포되었는지 확인합니다.

5. 새로운 포스트 작성 (Markdown 사용)

이제 블로그에 새로운 포스트를 추가하는 방법을 설명합니다. Markdown 파일을 사용해 쉽게 포스트를 작성할 수 있습니다.

5-1. Markdown 파일 생성

pages/posts 폴더에 새로운 Markdown 파일을 추가합니다. 예를 들어, test.md 파일을 다음과 같이 작성할 수 있습니다:

---
title: Test Article
date: 2024/9/20
description: It is a test article.
tag: test
author: You
---
 
It is a test article.
It is only for test purpose.

5-2. 포스트 빌드 및 배포

Markdown 파일을 추가한 후, 다시 블로그를 빌드하고 배포합니다:

npm run build
npm run deploy

이제 새로운 포스트가 블로그에 추가된 것을 확인할 수 있습니다.

6. 사이트맵

npm run build 수행 시마다 정적 사이트맵 파일(sitemap.xml)을 갱신 및 생성하게 하는 방법입니다.

6-1. next-sitemap 패키지 설치

npm install next-sitemap

6-2. next-sitemap.config.js 설정

프로젝트 루트에 next-sitemap.config.js 파일을 다음과 같이 생성합니다:

/** @type {import('next-sitemap').IConfig} */
module.exports = {
  siteUrl: process.env.SITE_URL || 'https://<your-github-username>.github.io',
  generateRobotsTxt: true,
  generateIndexSitemap: false,
}

6-3. package.json 수정

package.json 파일에 사이트맵 생성을 위한 스크립트를 추가합니다. scripts 섹션에 postbuild를 아래처럼 추가하세요:

{
  "scripts": {
    // ... 전략 ...
    "postbuild": "next-sitemap",
    // ... 후략 ...
  }
}

결론

Next.js Blog Boilerplate를 사용해 GitHub Pages에 블로그를 배포하는 방법을 알아보았습니다. Boilerplate의 적용, GitHub 리포지토리 생성 및 Git 설정, Next.js 설정, Markdown 포스트 작성, 사이트맵 생성에 이르는 방법을 Step-by-Step 설명하여, 순서대로 따라하면 손쉽게 정적 블로그를 만들어 낼 수 있음을 알 수 있습니다.
